VariableBean

Represents a variable definition within the system and supports cloning.

The VariableBean class encapsulates metadata about a variable, including its access level and type (e.g., a regular variable or a layout-related variable like view binding). This class implements CloneableBean for deep copying and Serializable for object serialization.

Access Modifiers:

  • ACCESS_MODIFIER_PRIVATE \u2013 private
  • ACCESS_MODIFIER_PROTECTED \u2013 protected
  • ACCESS_MODIFIER_PUBLIC \u2013 public

Variable Types:

  • VARIABLE \u2013 Standard variable
  • LAYOUT \u2013 Layout-based variable, e.g., view binding

Methods

getAllMetadata

Retrieves all metadata instances of the specified type from both requiredVariables and beanManifest.

This method iterates through all required variables and extracts metadata of the given type. It also retrieves matching metadata from the beanManifest, if available. This ensures that the returned list includes all metadata of the specified type, collected from all relevant data sources within the object.

The method handles null checks on beanManifest and its metadata collection to avoid NullPointerException, covering edge cases gracefully.

  • T: Unknown — the type of metadata to retrieve; must extend BeanMetadata

  • classType: Class<T> — the class object representing the metadata type to search for

  • return: a ArrayList<T extends <LinkRef name="BeanMetadata" />> of all metadata entries of the specified type, or an empty list if none found

cloneBean

Creates a deep copy of this VariableBean instance.

The method ensures that all primitive fields and mutable objects are duplicated properly to avoid shared references, thus preventing unintended side effects when modifying the cloned instance. Immutable or null-safe copies are made where applicable.

Specifically:

  • Strings are copied using new String(...) to ensure new references.
  • Booleans and Integers are boxed from primitives to ensure clarity and immutability.
  • DatatypeBean is cloned recursively using its cloneBean() method.
  • requiredVariables are deeply cloned, with each VariableBean copied individually.
  • icon byte array is copied using System.arraycopy.
  • variableTitles, variableValues, and inputType are shallow-copied using new HashMap<>(...).
  • beanManifest is assigned directly (TODO indicates deep cloning is intended in the future).
  • return: a new VariableBean instance with the same values and structure as this one

getDefCode

Generates the Java source code representation for this VariableBean, including all required (dependent) variables if present.

This method builds a complete variable declaration with modifiers (e.g., access, static, final), data type, variable name (or non-fixed formatted name), and optionally, an initializer expression. It uses values from variableValues to replace template keys in the initializer expression.

If requiredVariables are present, this method recursively appends their code declarations using the overloaded getDefCode(HashMap, VariableBean) method.

Template syntax keys such as %\{variableName} are replaced using the CodeFormatterUtils.getKeySyntaxString(String) utility.

  • return: the formatted Java code for the variable declaration(s)

getDefCode

Generates a single-line Java variable declaration for this VariableBean, formatted based on current modifiers, type, and name, and replaces any template syntax in the initializer using the provided variable values.

This method is internally used for generating code for nested or required variables by the main getDefCode() method.

Key behaviors include:

  • Modifier resolution via getAccessModifier().
  • Uses either variableName or a formatted nonFixedVariableName.
  • If the variable is globally initialized and has initializer code, it will substitute variable placeholders using variableValues.
  • Returns a complete declaration ending in a semicolon.

  • variableValues: String> — map of key-value replacements used for variable initializer template

  • parentVariableModel: VariableBean — the parent VariableBean, used for context (currently unused)

  • return: the generated Java variable declaration as a string
